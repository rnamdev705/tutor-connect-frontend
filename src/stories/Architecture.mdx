## Patterns

- **Server-side lists:** Cases, tutors, and invitations use API pagination, search, and filters (debounced ~300 ms).
- **Thin route pages:** `*View` client components own UI and data fetching.
- **Access control:** `proxy.ts` + `AppShell` + API authorization + role checks in views.
- **Forms:** `react-hook-form` + Zod schemas in `lib/validations/`.
- **Empty/error UI:** Shared `EmptyState`, `ErrorState`, and `getApiErrorMessage()` for safe toasts.
- **Storybook:** Global `withStoryProviders` decorator supplies TanStack Query; API-dependent modals use `withMockTutorsQuery` to seed cache.

## Protected route group

Authenticated pages live under `src/app/(app)/` and share `AppShell` (sidebar + top nav). Public routes: login, register, docs, unauthorized.

Tutors with incomplete profiles are redirected to `/profile/edit` by `TutorProfileGate`.
