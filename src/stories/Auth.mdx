## Session flow

1. User submits login/register (Zod + react-hook-form).
2. API returns JWT; stored in `localStorage` and mirrored to a cookie for `proxy.ts`.
3. `GET /auth/me` restores session on load.
4. `proxy.ts` guards protected routes on first navigation; `AppShell` guards after hydration.
5. On `401`, client clears token and redirects to `/login`.

## Route protection

| Layer | Mechanism |
|-------|-----------|
| **proxy.ts** | Cookie check → redirect to `/login?next=…` |
| **AppShell** | Client redirect if no authenticated user |
| **TutorProfileGate** | Incomplete tutors → `/profile/edit` |
| **API** | Authoritative access control on every endpoint |

## Feature-level gates

- **Parent only:** Create/edit cases, invite tutors, upload case documents.
- **Tutor only:** Invitation inbox, accept/decline, profile documents.
- **Owner check:** Parent must own the case to edit, invite, or delete.
