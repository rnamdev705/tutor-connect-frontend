# TutorConnect Frontend

Storybook documents shared UI components and patterns used across the TutorConnect tuition marketplace.

The **live app** talks to `tutorConnect-api` via an OpenAPI-generated client. Storybook documents UI components; stories that need data use inline `args` in the story file. API-backed stories (e.g. invite modals) require a running backend.

## Stack

- **Next.js 16** (App Router, `proxy.ts` route protection) + **React 19** + **TypeScript**
- **TanStack Query v5** — server-side pagination, cache, mutations
- **OpenAPI client** — `src/api/` (generated; run `npm run generate:api` after API changes)
- **Tailwind CSS v4** + **shadcn/ui**
- **Zod + react-hook-form** for auth validation

## Running locally

```bash
# Component catalog (most stories work offline; API-backed modals need tutorConnect-api)
npm run storybook   # http://localhost:6006

# Full app (API required)
cd ../tutorConnect-api && npm run dev   # :3001
cd ../tuition-marketplace && npm run dev   # :3000
```

Set `NEXT_PUBLIC_API_URL` in `.env.local` for the app. Storybook uses the same client setup via `@/api/setup`.

Browse **Common** in the sidebar for component stories. See **Guides** for architecture, auth, and routing.

Markdown docs also live in `docs/` and at `/docs` when the app is running.
